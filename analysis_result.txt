[535] 2.3 Проєктування структури бази даних
[536] Зберігає облікові записи користувачів з їхніми даними (email, пароль, ім'я, роль).
[537] TABLE REF: Таблиця 3.2 - Структура таблиці categories
[539] Ієрархічний каталог категорій товарів з підтримкою вкладеності через parentId.
[540] TABLE REF: Таблиця 3.3 - Структура таблиці characteristics
[542] Визначає типи характеристик (розмір, колір тощо), які можна призначати товарам певної категорії.
[543] TABLE REF: Таблиця 3.4 - Структура таблиці products
[545] TABLE REF: Основна таблиця товарів з інформацією про назву, ціну, кількість на складі та зображення.
[546] TABLE REF: Таблиця 3.5 - Структура таблиці product_characteristics
[548] TABLE REF: Зв'язкова таблиця між товарами та характеристиками, що зберігає конкретні значення.
[549] TABLE REF: Таблиця 3.6 - Структура таблиці reviews
[551] Відгуки користувачів на товари з оцінками, перевагами та недоліками.
[552] TABLE REF: Таблиця 3.7 - Структура таблиці orders
[554] Замовлення клієнтів з інформацією про доставку, оплату та статус.
[555] TABLE REF: Таблиця 3.8 - Структура таблиці order_items
[556] Позиції замовлень, що фіксують товари, їх кількість та ціну на момент покупки.
[557] TABLE REF: Таблиця 3.9 - Структура таблиці chats
[558] Чати підтримки між клієнтами та працівниками з прив'язкою до замовлень.
[559] TABLE REF: Таблиця 3.10 - Структура таблиці messages
[560] Повідомлення в чатах з підтримкою різних типів контенту (текст, зображення, файли).
